Plan: Replace Occupation Field with Persona Selection (Initial FSLSM Values)  

 Context

 The onboarding page currently has a mandatory occupation selectbox (Software Engineer, Data Scientist, etc.)
  that is concatenated into the learner_information string sent to the backend. Occupation is never parsed
 structurally — it's just free text the LLM sees when building the profile.

 We want to replace this with a persona picker — 5 learner archetypes, each with preset FSLSM dimension
 values. Selecting a persona sets the initial FSLSM baseline. The optional resume upload and text field
 remain, and the LLM can further refine the FSLSM values based on that additional information.

 No backend API schema changes are needed. The persona name + FSLSM values are included in the
 learner_information string. The only backend change is a prompt update to tell the LLM to use provided
 initial FSLSM values as a baseline.

 Persona Definitions (5 archetypes)
 ┌────────────────────┬────────────────────────┬─────────────────┬────────────────┬───────────────────┐
 │      Persona       │       Processing       │   Perception    │     Input      │   Understanding   │
 ├────────────────────┼────────────────────────┼─────────────────┼────────────────┼───────────────────┤
 │ Hands-on Explorer  │ -0.7 (active)          │ -0.5 (sensing)  │ -0.5 (visual)  │ -0.5 (sequential) │
 ├────────────────────┼────────────────────────┼─────────────────┼────────────────┼───────────────────┤
 │ Reflective Reader  │ 0.7 (reflective)       │ 0.5 (intuitive) │ 0.7 (verbal)   │ 0.5 (global)      │
 ├────────────────────┼────────────────────────┼─────────────────┼────────────────┼───────────────────┤
 │ Visual Learner     │ -0.2 (slightly active) │ -0.3 (sensing)  │ -0.8 (visual)  │ -0.3 (sequential) │
 ├────────────────────┼────────────────────────┼─────────────────┼────────────────┼───────────────────┤
 │ Conceptual Thinker │ 0.5 (reflective)       │ 0.7 (intuitive) │ 0.0 (balanced) │ 0.7 (global)      │
 ├────────────────────┼────────────────────────┼─────────────────┼────────────────┼───────────────────┤
 │ Balanced Learner   │ 0.0                    │ 0.0             │ 0.0            │ 0.0               │
 └────────────────────┴────────────────────────┴─────────────────┴────────────────┴───────────────────┘

 guidance.",
         "fslsm_dimensions": {"fslsm_processing": -0.7, "fslsm_perception": -0.5, "fslsm_input": -0.5,
 "fslsm_understanding": -0.5},
     },
     # ... etc
 }

 2. Update onboarding UI — frontend/pages/onboarding.py

 In render_information():

 showing the 5 persona names + short descriptions.
 - Store the selected persona name in st.session_state["learner_persona"] (replaces learner_occupation).
 - When building learner_information (line 153), prepend a structured string with the persona name and its
 FSLSM values, e.g.:
 "Learning Persona: Hands-on Explorer (initial FSLSM: processing=-0.7, perception=-0.5, input=-0.5,
 understanding=-0.5). " + resume text + user text
 - Update the "Save & Continue" validation (line 172) to check learner_persona instead of learner_occupation.

 3. Update session state — frontend/utils/state.py

 - Replace "learner_occupation" with "learner_persona" in PERSIST_KEYS (line 20) and
 initialize_session_state() (line 125).

 4. Update initialization prompt — backend/modules/adaptive_learner_modeling/prompts.py
